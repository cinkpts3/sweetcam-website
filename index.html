SweetCam GSoC 2025 Implementation Plan

Enhancing the IP Camera Honeypot with Multi-Protocol Support & Device Emulation

Project Overview

This project aims to transform SweetCam from a basic honeypot into a versatile, multi-protocol security tool capable of emulating more advanced IP camera functionalities. The goal is to enhance SweetCam with RTSP, ONVIF, and HTTP protocol support, containerize the solution with Docker, and create a template-based system for flexible emulation of various camera models.

Template-Based Device Emulation

  • YAML-based templates defining device metadata, protocols, and web interface features.
  • Inheritance support for templates, allowing easy reuse of base configurations.
  • JSON Schema validation for templates to ensure correctness and avoid errors.
  • CLI utilities like `sweetcam template validate` and `sweetcam template list` for managing templates.

New Protocol Support

RTSP Enhancements

  • Implement full RTSP state machine (RFC 2326).
  • Support for RTSP methods: DESCRIBE, SETUP, PLAY, TEARDOWN.
  • RTP/RTCP packet simulation, including static images, video files, and test patterns.

ONVIF Implementation

  • Implement WS-Discovery responder for LAN discovery of devices.
  • Provide core ONVIF services like Device Management, Media, and PTZ control.

Docker Usability Enhancements

  • Multi-stage Docker builds for efficient image creation.
  • Health checks to ensure the container is running properly.
  • Log rotation and auto-configuration generation to streamline setup.
  • Docker Compose profiles for various deployment setups.

HTTP Interface Improvements

  • Dynamic error generation (invalid login, session timeout, etc.).
  • Internationalization (i18n) with support for multiple languages.
  • Password recovery flow with fake email/SMS and lockout mechanisms.
