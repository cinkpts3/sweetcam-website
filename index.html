GSoC Implementation Plan: SweetCam

Project Overview

This project aims to transform SweetCam into a modular, multi-protocol honeypot simulating real-world IP cameras using ONVIF, RTSP, and customizable templates. We'll focus on flexibility, realism, and deployment simplicity.

Template-Based Device Emulation

  • YAML-based metadata: brand, firmware, network settings
  • Support for inheritance and JSON Schema validation
  • CLI tools: sweetcam template list, validate

New Protocol Support

RTSP: DESCRIBE, SETUP, PLAY, TEARDOWN; RTP/RTCP packets using images/video/test streams

ONVIF: WS-Discovery responder, Device, Media & PTZ services

Docker Enhancements

  • Multi-stage builds for smaller images
  • Docker health checks, log rotation
  • Compose profiles for dev/prod/test

HTTP Interface Improvements

  • Fake login, password reset, account lockout
  • Language toggle (i18n)
  • Session timeouts, login forms & error simulation
